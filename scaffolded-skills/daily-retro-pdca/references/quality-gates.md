# Quality Gates — Daily Retro PDCA

## Definition of Done

A cycle is complete when all of the following are true:

1. Prior cycle commitments reviewed: each assessed as done / partially done / not done, with a learning captured for each item
2. All available sources from the cycle period read: Slack, Gmail, Google Calendar, Granola, Obsidian vault
3. Human has reflected across at least two of the four lenses: interactions with people, leadership, software development, consulting/writing
4. Human has identified 1-3 achievable actions through Socratic dialogue — AI did not propose or suggest any action
5. Each action is specific (completion is knowable), achievable (fits within the next cycle period), and recorded where it will be found at the next retro
6. Human has explicitly closed the retro

## Common Failure Modes

Watch for these and treat them as quality failures requiring remediation:

- **Skipped data collection** — Insights generated before all available sources were read.
  Indicator: AI moved to Socratic questions before completing the data summary step.

- **Shallow action list** — Actions are vague, generic, or too large to complete in one cycle.
  Indicator: "be more present", "communicate better", "finish the project" — not specific enough to check.

- **AI-proposed actions** — AI named or suggested an action rather than asking a question.
  Indicator: any "you should", "consider doing", or "you could try" in the AI's output during DO or CHECK phase.

- **Prior commitments skipped** — New commitments made without reviewing last cycle's commitments first.
  Indicator: no accountability check completed in PLAN phase.

- **Laundry list** — More than 3 actions in the commitment list without explicit human request for more.
  Indicator: list exceeds 3 items.

- **Racing through reflection** — Retro completed too quickly to surface meaningful insights.
  Indicator: fewer than 2 lenses explored, or human did not pause to consider any question before answering.

## Gate Escalation

If a quality gate cannot be passed:
1. Identify the specific gap
2. Return to the relevant phase to address it
3. Do not reduce the quality bar to make the gate pass — fix the retro instead
