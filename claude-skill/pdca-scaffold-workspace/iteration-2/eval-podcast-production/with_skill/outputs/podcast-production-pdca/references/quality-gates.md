# Quality Gates — Podcast Production PDCA

## Definition of Done

A cycle is complete when all of the following are true:

1. Edited audio file is approved by the host and uploaded to the podcast host platform — confirmed by the host in writing
2. Show notes are published to the website, reviewed and approved by the host before publishing
3. LinkedIn post is scheduled (not yet sent) and has explicit host approval on the final version
4. Twitter/X thread is scheduled and has explicit host approval on the final version
5. Episode is live at the canonical episode URL and the guest has received a thank-you message with that link

## Common Failure Modes

Watch for these and treat them as quality failures requiring remediation:

- **AI copy losing the host's voice** — detectable when show notes or social posts use generic listicle language, avoid the host's characteristic sentence structures, or could have been written about any podcast guest. Remediate by returning to DO and revising with specific host-voice feedback.
- **Publishing without a host approval gate** — detectable when an asset goes live or is scheduled before a documented approval. Remediate by pausing all scheduling, pulling the asset if possible, and reinstating the gate before republishing.
- **Omitting or misattributing a key insight** — detectable by cross-checking each show notes takeaway against the approved insight list and the transcript timestamp. Remediate by correcting the show notes draft and re-presenting for host approval.
- **Social posts making claims not in the transcript** — detectable by verifying each factual claim and all quotes against the verbatim transcript. Remediate by correcting the draft; never publish an unverified claim.
- **Audio going out with a technical flaw** — detectable during the DO gate where the host reviews audio cue points. Remediate by flagging the segment, determining whether it is fixable in editing, and if not, determining whether the segment must be cut.

## Gate Escalation

If a quality gate cannot be passed:
1. Identify the specific gap (which check failed and why)
2. Return to DO phase to address it — do not declare done with the gap open
3. Do not reduce the quality bar to make the gate pass — fix the work instead
4. If the gap is caused by a process violation (a gate was skipped), document it in the ACT phase retrospective and propose a skill refinement to prevent recurrence
