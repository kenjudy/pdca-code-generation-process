# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## What This Repo Is

A skill factory for human-supervised AI collaboration using Plan-Do-Check-Act (PDCA) methodology. It contains two first-class skills:

- **pdca-framework** — TDD-disciplined software development PDCA cycle. Master prompt files in `1. Plan/`, `2. Do/`, `3. Check/`, `4. Act/`. Built into `pdca-framework.skill`.
- **pdca-scaffold** — Generalized PDCA skill generator. Uses Socratic questioning to produce a domain-specific PDCA skill for any complex repeatable human task. Master prompt files in `5. Scaffold/`. Built into `pdca-scaffold.skill`.

## Commands

**Unit tests (no API key needed):**
```bash
cd claude-skill && bash run-tests.sh
```
Builds the skill, runs ruff lint, runs pytest. `WARN_ONLY=1 bash run-tests.sh` for git hook mode (never blocks).

**Run a single test class:**
```bash
cd claude-skill && uv run python -m pytest tests/test_build.py -v
```

**Lint and type check:**
```bash
cd claude-skill && uv run ruff check .
cd claude-skill && uv run mypy eval tests
```

**Build pdca-framework skill:**
```bash
bash claude-skill/build-skill.sh
```
Reads master prompt files, generates `claude-skill/pdca-framework/references/`, zips into `claude-skill/pdca-framework.skill`.

**Build pdca-scaffold skill:**
```bash
bash claude-skill/build-scaffold.sh
```
Reads `5. Scaffold/*.md`, generates `claude-skill/pdca-scaffold/references/`, zips into `claude-skill/pdca-scaffold.skill`.

**Install locally after build:**
```bash
unzip -o claude-skill/pdca-framework.skill -d ~/.claude/skills/
unzip -o claude-skill/pdca-scaffold.skill -d ~/.claude/skills/
```

**Eval tests (requires `ANTHROPIC_API_KEY`, ~$2–5/run, manual only):**
```bash
cd claude-skill && uv sync --extra eval
bash run-evals.sh                                              # all evals
bash run-evals.sh tests/test_evals.py::TestPrompt2Evals       # single class
```
Note: use `tests/test_evals.py::ClassName` syntax, not `-k` — `-k` exits with code 5 (no tests collected).

## Architecture

```
Master Prompts (Obsidian .md)  →  build-skill.sh     →  pdca-framework.skill
1. Plan/1a...md                                           ↓
1. Plan/1b...md              →  pdca-framework/references/ →  .skill zip
2. Do/2...md                                              ↓
3. Check/3...md                                     Install to ~/.claude/skills/
4. Act/4...md                                       or upload to claude.ai
Human Working Agreements.md

5. Scaffold/5a...md          →  build-scaffold.sh   →  pdca-scaffold.skill
5. Scaffold/5b...md                                       ↓
5. Scaffold/5c...md          →  pdca-scaffold/references/ →  .skill zip
```

**Never edit `claude-skill/pdca-framework/references/` or `claude-skill/pdca-scaffold/references/` directly** — they are build artifacts regenerated by their respective build scripts.

**Eval pipeline:** `tests/test_evals.py` sends each phase prompt + a scenario to Claude Haiku as LLM-as-judge, scoring against rubrics in `eval/rubrics/`. Results written to `eval/results/`.

## Key Files

| File | Role |
|---|---|
| `1. Plan/1a...md` – `4. Act/...md` | pdca-framework master prompt content (source of truth) |
| `5. Scaffold/5a...md` – `5c...md` | pdca-scaffold master prompt content (source of truth) |
| `claude-skill/pdca-framework/SKILL.md` | pdca-framework skill descriptor (manually maintained) |
| `claude-skill/pdca-scaffold/SKILL.md` | pdca-scaffold skill descriptor (manually maintained) |
| `claude-skill/build-skill.sh` | Assembles pdca-framework.skill from masters |
| `claude-skill/build-scaffold.sh` | Assembles pdca-scaffold.skill from masters |
| `claude-skill/eval/rubrics/rubric_*.py` | LLM-as-judge criteria per phase |
| `claude-skill/eval/scenarios/*.json` | Test inputs for each phase |
| `claude-skill/tests/test_build.py` | Validates ZIP structure, description length, file contents |
| `.github/workflows/test.yml` | CI: ruff + mypy + build-and-test on push/PR |
| `.github/workflows/release.yml` | Triggered by `v*.*.*` tag push; runs tests, builds, publishes GitHub Release |

## Releasing

1. Update version in `claude-skill/README.md`
2. Note change in `claude-skill/PDCA-SKILL-CHANGELOG.md`
3. Commit and push to main
4. `git tag v1.x.x && git push --tags`

The release workflow runs automatically and attaches `pdca-framework.skill` to the GitHub Release.

## Issue Tracking (Beads)

This project uses **beads (`bd`)** for issue tracking. Run `bd onboard` to get started.

```bash
bd ready                              # Find available work
bd show <id>                          # View issue details
bd update <id> --status in_progress   # Claim work
bd close <id>                         # Complete work (after human sign-off)
bd sync                               # Sync with git
```

## Session Completion (Mandatory)

Work is **not complete** until `git push` succeeds and the human has signed off on the Check and Act phases.

**Required sequence:**
1. File beads issues for any remaining work
2. Run quality gates if code changed: `cd claude-skill && bash run-tests.sh`
3. Close finished issues, update in-progress ones
4. Push:
   ```bash
   git pull --rebase
   bd sync
   git push
   git status  # must show "up to date with origin"
   ```
5. Hand off — provide context for next session

**NEVER** say "ready to push when you are" — push yourself. If push fails, resolve and retry.

## Planning Discipline

When planning any new skill or build artifact, the implementation plan must include
explicit tasks for build system, documentation, CHECK criteria, and ACT retrospective.
Don't assume them — name them in the plan. See the 1b prompt for the full checklist.

## Human Supervision (Non-Negotiable)

See `claude-skill/Agent.md` for the full protocol. Key rules:

- **STOP and present to the human** before writing code, committing, pushing, or closing issues
- All code changes require a failing test first (TDD, no exceptions)
- Do not batch multiple steps without human confirmation between them

## Validating Prompt Changes

Any edit to master prompt files or eval rubrics requires:

1. Check `claude-skill/eval/results/` for the baseline scores
2. Make one discrete change, rebuild (`bash claude-skill/build-skill.sh`)
3. Re-run only the affected phase evals
4. All previously passing scenarios must still pass — see `claude-skill/Agent.md` for the full table of phase → eval class mappings
