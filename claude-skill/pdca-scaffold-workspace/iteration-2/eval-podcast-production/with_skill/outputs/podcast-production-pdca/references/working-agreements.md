# Working Agreements — Podcast Production PDCA

**Version:** 1
**Last refined:** 2026-05-05
**Refinement history:** See git log for this file.

## Process Discipline

Process discipline trumps speed. When a violation is observed, stop and address it
before continuing.

### Human Owns (Non-Negotiable)

- Selecting the 3-5 key insights to surface from the interview — editorial framing cannot be delegated
- Approving editorial voice and tone in all show notes and social posts before they proceed
- Deciding whether any audio segment is broadcast-quality or must be cut
- Approving the episode title and description (SEO and brand positioning)
- Final sign-off before any asset is published or scheduled
- Sending guest quote-approval requests — these set external expectations with a named guest

### AI Executes (With Review Gates)

- Transcribing raw audio — AI transcribes; host reviews transcript before insight-selection gate
- Drafting show notes from transcript (summary, guest bio pull, key takeaways, links) — AI drafts, host approves before publishing
- Drafting LinkedIn post — AI drafts, host approves before scheduling
- Drafting Twitter/X thread — AI drafts, host approves before scheduling
- Generating chapter markers and timestamps — AI generates, host reviews before uploading
- Formatting and proofreading show notes — AI executes, host reviews in approval package
- Creating per-episode publishing checklist — AI generates, used by host to verify completion

### Intervention Triggers

When the AI does any of the following, stop immediately:

- Stop if AI publishes or schedules any asset without explicit host approval
- Stop if AI drafts social content that includes a direct quote not verbatim from the transcript
- Stop if AI selects the key insights or frames the episode angle without host confirmation
- Intervene when AI expands scope to unplanned platforms or content types not in the approved plan
- Intervene when AI begins drafting any asset before the transcript review and insight-selection gate is complete

### Process Debt to Watch For

These shortcuts tend to accumulate — flag them when they appear:

- Skipping the transcript review to "save time" and going straight to drafting
- Reusing a social post template so often that every episode sounds identical
- Host approvals becoming rubber stamps (approving without actually reading the draft)
- Letting audio quality bar drift downward ("good enough" creep episode over episode)
- Not updating the show notes template when the podcast format or segment structure evolves

## Implementation Approach

- Understand context before executing. Review the transcript and confirm the episode angle
  before beginning any drafting. "Analysis done" must be true before moving to DO.
- Minimal scope. The smallest action that moves the cycle forward. No additional platforms,
  content formats, or promotion tasks without explicit host approval.
- One phase at a time. Complete PLAN before DO. Complete DO before CHECK.
- Respond to feedback. If an AI draft is missing the host's voice, revise using the feedback
  as a style signal — do not ask the host to lower their standards.

## Definition of Done

A cycle is complete only when all of the following are true:

- [ ] Edited audio file is approved by the host and uploaded to the podcast host platform
- [ ] Show notes are published to the website, reviewed and approved by the host
- [ ] LinkedIn post is scheduled (not yet sent) and approved by the host
- [ ] Twitter/X thread is scheduled and approved by the host
- [ ] Episode is live and the guest has received a thank-you message with the episode link
